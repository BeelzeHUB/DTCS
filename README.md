## 📌 Project Description

**Development and Implementation of a Hospital Information Management System (HIMS) with Virtual MediSense AI** is a web-based healthcare platform designed to modernize hospital operations by digitizing clinical and administrative workflows. The system integrates a comprehensive set of hospital management modules with **Virtual MediSense AI**, an *Intelligent Clinical Decision Support Assistant* that augments the capabilities of healthcare professionals.

The platform replaces manual, paper-based, and fragmented hospital processes with a unified, secure, and intelligent digital ecosystem. It streamlines patient management, diagnostics, pharmacy, radiology, surgery scheduling, dietetics, billing, and reporting — while providing AI-assisted clinical insight at the point of care.

### Virtual MediSense AI

Virtual MediSense AI assists healthcare professionals in four core clinical domains:

| Domain | Description |
|--------|-------------|
| 🩺 **Symptom Assessment** | Structures patient-reported symptoms and suggests possible clinical pathways for review. |
| 🔬 **Diagnostic Assistance** | Surfaces relevant differential considerations and recommends appropriate diagnostic tests. |
| 💊 **Treatment Recommendation** | Proposes evidence-informed treatment options aligned with patient context. |
| 🏥 **Clinical Service Support** | Helps coordinate clinical services, referrals, and care-plan workflows. |

> ⚠️ **Important Disclaimer:** Virtual MediSense AI provides *recommendations only*. All outputs are intended to **support** licensed healthcare professionals and must **never replace professional medical judgment**. The final clinical decision rests solely with the attending physician or authorized clinician.

## 🎯 Project Objectives

### 🌟 General Objective

To design, develop, and implement a web-based Hospital Information Management System integrated with Virtual MediSense AI that digitizes hospital operations and delivers intelligent clinical decision support for symptom assessment, diagnostic assistance, treatment recommendation, and clinical service management.

### 🔹 Specific Objectives

1. Develop a secure user authentication and **role-based access control** system for all hospital personnel.
2. Implement a centralized **administrative dashboard** providing real-time operational insights.
3. Build a **Patient Management** module for registration, demographics, and medical history.
4. Create an **Electronic Medical Records (EMR)** system to digitize and secure patient clinical data.
5. Develop **Appointment Scheduling** to optimize clinician and facility utilization.
6. Implement **Billing and Payments** for accurate financial transactions and invoicing.
7. Integrate **Notifications** to keep stakeholders informed of critical events.
8. Generate **Reports and Analytics** for data-driven hospital management.
9. Maintain **Audit Logs** for traceability, compliance, and accountability.
10. Develop dedicated clinical modules: **LIS, RIS, PMS, SORS, and DNMS**.
11. Integrate **Virtual MediSense AI** powered by the Google Gemini API for clinical decision support.
12. Connect external APIs (**Google Maps, Zoom, Zapier**) to extend system capabilities.
13. Ensure a responsive, accessible, and modern user interface across devices.

## 🔭 Scope of the System

### ✅ In Scope

- Web-based hospital administration and clinical workflow management.
- Patient registration, EMR, appointments, billing, and reporting.
- Five specialized clinical modules (LIS, RIS, PMS, SORS, DNMS).
- AI-assisted clinical decision support via Virtual MediSense AI.
- Role-based multi-user access with audit logging.
- Integration with external APIs (Gemini, Maps, Zoom, Zapier).

### ❌ Out of Scope / Limitations

- The system does **not** provide autonomous medical diagnosis or treatment.
- Hardware/ biomedical device drivers are not implemented.
- Telemedicine video infrastructure is delegated to the Zoom API integration.
- Native mobile applications are planned as future enhancements (not in v1.0.0).
- The AI model is not fine-tuned on private hospital data; it relies on general medical knowledge via the Gemini API.

### 🔬 Clinical Modules

#### Laboratory Information System (LIS)

- **Laboratory Requests** — Order and track lab tests.
- **Sample Collection** — Manage specimen barcodes and collection status.
- **Test Processing** — Record results from analyzers and manual entry.
- **Laboratory Results** — View and validate patient results.
- **Result Verification** — Dual-control sign-off workflow.
- **Laboratory Reports** — Generate printable result documents.

#### Radiology and Imaging System (RIS)

- **Imaging Requests** — Order X-ray, CT, MRI, and ultrasound studies.
- **Scheduling** — Book imaging slots and rooms.
- **Radiologist Assignment** — Assign studies to radiologists.
- **Imaging Reports** — Dictate and finalize radiology reports.
- **Image History** — Longitudinal view of patient imaging.

#### Pharmacy Management System (PMS)

- **Medicine Inventory** — Catalog drugs, strengths, and locations.
- **Stock Monitoring** — Track stock levels and low-stock alerts.
- **Dispensing** — Fill prescriptions and reduce inventory.
- **Prescription Validation** — Verify authenticity and interactions.
- **Expiration Monitoring** — Flag near-expiry medicines.
- **Purchase Orders** — Replenish stock from suppliers.

#### Surgery and Operating Room Scheduler (SORS)

- **Surgery Scheduling** — Plan elective and emergency surgeries.
- **Operating Room Reservation** — Reserve ORs by time slot.
- **Surgeon Assignment** — Assign surgical teams.
- **Equipment Scheduling** — Reserve surgical instruments and equipment.
- **Surgical Records** — Document procedures and outcomes.

#### Diet and Nutrition Management System (DNMS)

- **Diet Prescription** — Prescribe therapeutic diets per patient.
- **Meal Planning** — Build daily meal schedules.
- **Nutrition Monitoring** — Track intake vs. dietary restrictions.
- **Dietary Recommendations** — Suggest meals within restrictions.
- **Nutrition Reports** — Summarize nutritional status.

## 🤖 Virtual MediSense AI

Virtual MediSense AI is an *Intelligent Clinical Decision Support Assistant* embedded within the HIMS. It is **powered by the Google Gemini API** and provides contextual, AI-generated insight to clinical staff.

| Capability | Description |
|------------|-------------|
| 🩺 **Symptom Assessment** | Structures chief complaints and suggests clinical pathways. |
| 🔬 **Diagnostic Assistance** | Recommends differential diagnoses and relevant tests. |
| 💊 **Treatment Recommendation** | Proposes evidence-informed treatment plans. |
| 🏥 **Clinical Service Support** | Coordinates referrals, consults, and care workflows. |
| 💬 **AI Chat Assistant** | Conversational interface for clinical queries. |
| 📚 **Medical Knowledge Assistance** | Retrieves general medical explanations and references. |
| 🧠 **Clinical Decision Support** | Surfaces decision aids at the point of care. |
| 🤝 **Patient Inquiry Support** | Answers patient education questions safely. |

> 🔒 All AI responses are clearly labeled as **advisory**. Licensed professionals retain full authority over clinical decisions.

## 🏗️ System Architecture

The system follows a **layered MVC architecture** with a decoupled frontend, Laravel backend, MySQL database, and external API integrations.

| Layer | Technology | Responsibility |
|-------|-----------|----------------|
| **Frontend** | HTML5, CSS3, Bootstrap 5, JS | UI rendering & interactivity |
| **Backend** | Laravel (PHP) | Business logic, routing, controllers |
| **Database** | MySQL | Persistent data storage |
| **REST API** | Laravel routes/controllers | Client-server communication |
| **AI Integration** | Google Gemini API | Clinical decision support |
| **External APIs** | Maps, Zoom, Zapier | Location, telehealth, automation |

## ⚙️ Installation Guide

### 1. Clone the Repository
```bash
git clone https://github.com/BeelzeHUB/DTCS.git
cd DTCS
```

### 2. Install PHP Dependencies
```bash
composer install
```

### 3. Install Frontend Dependencies
```bash
npm install
```

### 4. Configure Environment
```bash
cp .env.example .env
```

### 5. Generate Application Key
```bash
php artisan key:generate
```

### 6. Configure Database
Edit `.env` (see [Database Configuration](#️-database-configuration)).

### 7. Run Migrations
```bash
php artisan migrate
```

### 8. Seed Database
```bash
php artisan db:seed
```

### 9. Link Storage
```bash
php artisan storage:link
```

### 10. Run Laravel Server
```bash
php artisan serve
```

### 11. Run Vite (Frontend)
```bash
npm run dev
```

### 12. Deploy with Laragon
Open **Laragon** → Start *Apache* + *MySQL* → point the document root to `public/`.

### 13. Deploy Frontend with Vercel
```bash
vercel --prod
```

DB_PORT=3306

DB_PASSWORD=

## 🛠️ Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| `composer install` fails | Missing PHP extensions | Enable `mbstring`, `openssl`, `pdo_mysql`, `gd`. |
| `npm install` errors | Node version mismatch | Use Node 18+ LTS. |
| `php artisan key:generate` error | Missing `.env` | Run `cp .env.example .env`. |
| Migration fails | Wrong DB credentials | Check `.env` DB settings. |
| 500 Server Error | Cache/permission | `php artisan config:clear && storage:chmod`. |
| Gemini API not responding | Invalid key | Verify `GEMINI_API_KEY` in `.env`. |
| Vercel build fails | Missing build script | Ensure `npm run build` works locally. |
| Git push rejected | Out-of-date branch | `git pull --rebase` then `git push`. |
| Laragon not serving | Wrong doc root | Point to `public/`. |
| MySQL won't start | Port conflict | Free port 3306 or change config. |
